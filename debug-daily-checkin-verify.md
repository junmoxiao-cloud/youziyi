# [OPEN] daily-checkin-verify

## 目标

- 启动后端与 Web。
- 验证每日打卡提交流程。
- 验证 `checkin-status`、`daily` 聚合接口。
- 验证 Web 访问链路是否正常。
- 若发现问题，仅修复正式仓库，并同步更新 `tasks/checklist`。

## 可证伪假设

1. 后端未监听约定端口或环境变量不一致，导致 Web 或接口访问失败。
2. 每日打卡提交接口请求体或字段映射与前端表单不一致，导致提交失败或数据未落库。
3. `checkin-status` 或 `daily` 聚合接口的响应包装层发生变化，导致 Web 端读取路径错误。
4. Web 端代理或 API 基地址仍指向旧端口/旧主机，导致访问链路异常。
5. 每日打卡成功后状态回写不完整，导致 `checkin-status` 与页面展示不一致。

## 计划

1. 确认正式仓库结构与启动方式。
2. 启动后端与 Web，收集运行输出。
3. 使用本地请求与页面访问验证关键接口和提交流程。
4. 若发现异常，先定位证据，再做最小修复。
5. 更新 `tasks/checklist` 并整理验证结果。
