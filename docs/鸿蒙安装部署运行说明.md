# 鸿蒙端安装部署运行说明

本文档面向评审人复现部署，描述《游子衣》鸿蒙端（`apps/harmony`）从环境准备、依赖安装、编译构建、签名部署到运行验证的完整流程。所有版本号、SDK 版本、端口、域名、权限名与配置项均取自工程真实配置文件，未做虚构。

> 工程根目录：`c:\Users\dou12\Desktop\youziyi\youziyi-oss\apps\harmony`
> 应用包名（`bundleName`）：`com.youziyi.harmony`（取自 `AppScope/app.json5`）
> 应用版本：`versionName = 1.0.0`，`versionCode = 1000000`

---

## 1. 环境要求

### 1.1 DevEco Studio

- 建议安装最新稳定版 **DevEco Studio**（需支持 HarmonyOS NEXT / API 12+），用于打开、编译、签名、烧录与调试本工程。
- DevEco Studio 内置 HarmonyOS SDK 与 hvigor 构建工具链，安装后请通过 `File > Settings > SDK Manager` 确认已下载与 `compatibleSdkVersion`、`targetSdkVersion` 对应的 SDK 组件。

### 1.2 HarmonyOS SDK 版本

以下版本取自 `apps/harmony/build-profile.json5` 中 `products.default` 的真实配置，本机已安装的 SDK 必须能够覆盖：

| 配置项 | 真实值 | 说明 |
| --- | --- | --- |
| `compatibleSdkVersion` | `5.0.0(12)` | 兼容的最低 HarmonyOS SDK 版本（API 12） |
| `targetSdkVersion` | `6.0.2(22)` | 目标 HarmonyOS SDK 版本（API 22） |
| `runtimeOS` | `HarmonyOS` | 运行时为 HarmonyOS |
| `buildModeSet` | `debug`、`release` | 工程支持的两种构建模式 |

> 若本机 SDK 版本低于 `5.0.0(12)`，DevEco Studio 打开工程时会报 SDK 缺失或版本不匹配，请先在 SDK Manager 中下载对应版本后再继续。

### 1.3 支持的设备类型

取自 `apps/harmony/entry/src/main/module.json5` 中 `module.deviceTypes`：

- `phone`（手机）
- `tablet`（平板）
- `2in1`（二合一设备）

部署时请选择上述三类设备之一，其他设备类型不在本工程支持范围内。

### 1.4 Node.js / JDK / HDC 工具

- **Node.js**：建议安装 LTS 版本（如 18.x 或 20.x），供 hvigor 构建脚本与依赖管理使用。
- **JDK**：DevEco Studio 自带运行所需的 JDK，一般无需单独配置；如需命令行调用 hvigor，请确保 `JAVA_HOME` 已正确指向 DevEco Studio 内置 JDK 或本机 JDK 17+。
- **HDC（HarmonyOS Device Connector）**：随 DevEco Studio 一并安装，用于真机调试、安装 HAP、查看日志。命令行使用前请确认 `hdc` 已加入系统 `PATH`（DevEco Studio 安装目录下的 `toolchains/hdc`）。

## 2. 依赖安装

### 2.1 oh-package.json5 说明

工程根目录 `apps/harmony/oh-package.json5` 内容如下（`modelVersion` 为 `5.0.0`）：

```json5
{
  "modelVersion": "5.0.0",
  "description": "Harmony frontend skeleton for YouZiYi.",
  "dependencies": {},
  "devDependencies": {
    "@ohos/hypium": "1.0.19",
    "@ohos/hamock": "1.0.0"
  }
}
```

- **dependencies**：当前为空，第一阶段业务代码内聚在 `entry` 模块中，暂未引入额外运行期三方依赖。
- **devDependencies**：
  - `@ohos/hypium` 版本 `1.0.19`：HarmonyOS 单元测试框架，供 `*.test.ets` 用例使用。
  - `@ohos/hamock` 版本 `1.0.0`：HarmonyOS Mock 工具，用于在测试中对系统接口进行模拟。

`entry` 模块下的 `apps/harmony/entry/oh-package.json5` 同样为最小配置（`name = entry`、`version = 1.0.0`），无额外依赖。

### 2.2 oh_modules 安装

依赖产物目录为 `apps/harmony/oh_modules` 与 `apps/harmony/entry/oh_modules`。推荐两种方式：

- **DevEco Studio 自动同步**：用 DevEco Studio 打开 `apps/harmony` 工程后，IDE 会自动执行 `ohpm install`；若未自动触发，可在工程右键菜单选择 `Sync and Refresh Project`，或在 DevEco Studio 终端执行：

  ```bash
  ohpm install
  ```

- **命令行手动安装**：在 `apps/harmony` 目录下执行：

  ```bash
  ohpm install
  ```

  安装完成后，`oh_modules` 目录会被生成，`@ohos/hypium` 与 `@ohos/hamock` 会出现在其中。

### 2.3 hvigor 配置说明

- `apps/harmony/hvigor/hvigor-config.json5`：`modelVersion = 5.0.0`，未声明额外依赖与执行参数，使用 hvigor 默认配置。
- `apps/harmony/hvigorfile.ts`：引入 `@ohos/hvigor-ohos-plugin` 的 `appTasks`，作为 App 级构建任务入口：

  ```ts
  import { appTasks } from '@ohos/hvigor-ohos-plugin';

  export default {
    system: appTasks,
    plugins: []
  }
  ```

- `apps/harmony/entry/hvigorfile.ts`：引入同一插件的 `hapTasks`，作为 HAP 模块构建任务入口：

  ```ts
  import { hapTasks } from '@ohos/hvigor-ohos-plugin';

  export default {
    system: hapTasks,
    plugins: []
  }
  ```

如需新增自定义构建插件，请在对应 `hvigorfile.ts` 的 `plugins` 数组中追加，第一阶段不建议改动。

## 3. 编译步骤

### 3.1 构建模式

`build-profile.json5` 中 `buildModeSet` 声明了两种构建模式：

| 模式 | 用途 |
| --- | --- |
| `debug` | 本地调试、真机/模拟器热重载与日志输出 |
| `release` | 正式发布、签名打包、上传分发 |

### 3.2 DevEco Studio 构建

1. 用 DevEco Studio 打开 `apps/harmony` 目录。
2. 在工具栏选择 `Build > Build Hap(s)/APP(s) > Build Hap(s)` 进行 debug 构建。
3. 若需 release 产物，先在 `Run > Edit Configurations` 或 `Build` 菜单中将 `Build Mode` 切换为 `release`，再执行 `Build Hap(s)`。
4. 构建产物默认输出在 `apps/harmony/entry/build/default/outputs/default/`（debug）或对应 release 目录下，文件后缀为 `.hap`。

### 3.3 hvigor 命令行构建

在 `apps/harmony` 目录下可使用 hvigor 命令行构建：

```bash
# debug 构建
hvigorw assembleHap --mode module -p product=default -p buildMode=debug --no-daemon

# release 构建
hvigorw assembleHap --mode module -p product=default -p buildMode=release --no-daemon
```

构建参数说明：

- `--mode module`：以模块方式构建 HAP。
- `-p product=default`：使用 `build-profile.json5` 中 `products` 里名为 `default` 的产品（其 `compatibleSdkVersion = 5.0.0(12)`、`targetSdkVersion = 6.0.2(22)`、`signingConfig = default`）。
- `-p buildMode=debug|release`：与 `buildModeSet` 中声明的模式对应。

### 3.4 产物 HAP 说明

- 产物为单个 `entry-default-signed.hap`（已签名）或 `entry-default-unsigned.hap`（未签名），位于 `entry/build/default/outputs/default/` 目录下。
- HAP 内含 ArkTS 编译产物、`resources`、`module.json5` 与 `app.json5` 配置，安装时由系统解析权限与入口 Ability。

## 4. 部署流程

### 4.1 签名配置说明

`apps/harmony/build-profile.json5` 中 `app.signingConfigs` 配置如下（已脱敏处理密钥口令，请以本机实际为准）：

```json5
{
  "signingConfigs": [
    {
      "name": "default",
      "type": "HarmonyOS",
      "material": {
        "certpath": "C:\\Users\\dou12\\.ohos\\config\\default_harmony_xxx.cer",
        "keyAlias": "debugKey",
        "keyPassword": "<本机生成的密钥口令>",
        "profile": "C:\\Users\\dou12\\.ohos\\config\\default_harmony_xxx.p7b",
        "signAlg": "SHA256withECDSA",
        "storeFile": "C:\\Users\\dou12\\.ohos\\config\\default_harmony_xxx.p12",
        "storePassword": "<本机生成的密钥库口令>"
      }
    }
  ]
}
```

字段含义：

| 字段 | 含义 |
| --- | --- |
| `name` | 签名方案名，`products.default.signingConfig` 引用此名称 |
| `type` | `HarmonyOS`，表示 HarmonyOS 应用签名 |
| `material.certpath` | 调试/发布证书文件（`.cer`）路径 |
| `material.keyAlias` | 密钥别名，本工程为 `debugKey` |
| `material.keyPassword` | 密钥口令（加密存储） |
| `material.profile` | Profile 文件（`.p7b`）路径 |
| `material.signAlg` | 签名算法，固定为 `SHA256withECDSA` |
| `material.storeFile` | 密钥库文件（`.p12`）路径 |
| `material.storePassword` | 密钥库口令（加密存储） |

> **重要安全提示**：
> 1. 上述 `certpath`、`profile`、`storeFile` 路径为开发者本机生成路径，**评审人请使用本机 DevEco Studio 重新生成的调试证书与 Profile**，禁止直接复用仓库中他人证书或硬编码密钥口令。
> 2. 推荐通过 DevEco Studio 的 `File > Project Structure > Signing Configs` 勾选 `Automatically generate signature` 让 IDE 自动为本机生成调试签名并回填 `build-profile.json5`。
> 3. 仓库提交时，请勿将含有真实密钥口令的 `build-profile.json5` 推送到公开分支；如需团队共享，应将签名配置改为占位并通过本地未跟踪文件注入。

### 4.2 真机安装

1. **开发者模式**：在 HarmonyOS 真机「设置 > 关于本机 > 软件版本」连续点击 7 次开启开发者模式，再进入「设置 > 系统与更新 > 开发者选项」打开「USB 调试」与「仅充电模式下允许 ADB 调试」等选项。
2. **连接设备**：通过 USB 连接真机与电脑，执行以下命令确认设备已识别：

   ```bash
   hdc list targets
   ```

   设备序列号出现即表示连接成功。

3. **安装 HAP**：

   ```bash
   # 安装已签名的 debug HAP
   hdc install entry/build/default/outputs/default/entry-default-signed.hap
   ```

   安装成功后会输出 `AppBundle install success`，并在真机桌面生成「游子衣」应用图标。
4. **DevEco Studio 直装**：也可在 DevEco Studio 中选择真机后点击 `Run`，IDE 会自动完成构建、签名、安装与启动 EntryAbility。

### 4.3 模拟器安装

1. 在 DevEco Studio 中打开 `Tools > Device Manager`，创建或选择一台 HarmonyOS 模拟器（设备类型需为 `phone`、`tablet` 或 `2in1` 之一，与 `module.json5` 中 `deviceTypes` 一致）。
2. 启动模拟器后，在运行设备列表中选中该模拟器，点击 `Run` 即可自动安装并运行。
3. 模拟器场景下通常使用 IDE 自动生成的调试签名，无需额外配置发布证书。

## 5. 运行方法

### 5.1 启动入口

应用入口 Ability 为 `EntryAbility`，相关定义取自 `apps/harmony/entry/src/main/module.json5`：

- `module.mainElement = EntryAbility`
- `abilities[0].name = EntryAbility`
- `abilities[0].srcEntry = ./ets/entryability/EntryAbility.ets`
- `skills` 配置了 `entity.system.home` 与 `action.system.home`，即点击桌面图标直接启动该 Ability。

### 5.2 首次启动流程

页面壳位于 `apps/harmony/entry/src/main/ets/pages/`，首次启动遵循以下流转：

```text
Splash -> Login -> RoleSelect -> Onboarding -> FamilyJoin -> Main
```

- **Splash**：启动闪屏与初始化。
- **Login**：登录/注册，登录页会先请求 `GET /api/health` 确认鸿蒙端已连接服务端。
- **RoleSelect**：角色选择（`elder` / `child`）。
- **Onboarding**：首次信息采集（城市码 `cityCode`、追踪指标 `trackedMetrics`）。
- **FamilyJoin**：家庭连接（创建家庭获取陪伴码或输入陪伴码加入家庭）。
- **Main**：主面板，承载四个底部标签页。

### 5.3 四个底部标签页

`Main` 页面承载以下四个底部标签页（取自 `apps/harmony/entry/src/main/ets/pages/tabs/`）：

| 标签页文件 | 标签 |
| --- | --- |
| `HomeTab.ets` | 首页 |
| `InteractionTab.ets` | 互动 |
| `HealthTab.ets` | 健康（每日打卡入口） |
| `ProfileTab.ets` | 我的（账号资料与家庭基础信息） |

### 5.4 远程 / Mock 切换方式

切换逻辑全部集中在 `apps/harmony/entry/src/main/ets/common/AppConfig.ets`，默认配置如下：

```ets
const CLOUD_API_BASE_URL: string = 'https://api.youziyi.com'
const LOCAL_API_BASE_URL: string = 'http://localhost:3001'
const DEFAULT_APP_ENVIRONMENT: AppEnvironment = AppEnvironment.PRODUCTION

export const APP_CONFIG: AppConfig = {
  environment: DEFAULT_APP_ENVIRONMENT,
  apiBaseUrl: resolveDefaultApiBaseUrl(DEFAULT_APP_ENVIRONMENT),
  apiMode: ApiMode.REMOTE,
  verification: resolveDefaultVerificationConfig(DEFAULT_APP_ENVIRONMENT)
}
```

环境与接口基址绑定关系：

| `environment` | `apiBaseUrl` |
| --- | --- |
| `production`（默认） | `https://api.youziyi.com` |
| `development` | `http://localhost:3001` |

`verification` 配置（仅开发态生效）：

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `requestMode` | `ApiMode.REMOTE` | 验证码请求模式，登录页会先请求 `GET /api/health` 做服务端连通确认 |
| `allowMockFallback` | `true`（仅 development） | 远程失败后是否回退 Mock；正式环境下强制不生效 |
| `enableUniversalCode` | `true`（仅 development） | 是否启用万能验证码；正式环境下强制不生效 |
| `universalCode` | `'123456'` | 万能验证码，仅开发态可用 |
| `remoteBindingPath` | `'/api/health'` | 远程连通确认路径 |

常用切换方式（修改 `AppConfig.ets` 后重新构建生效）：

- **全量切到 Mock**：将 `APP_CONFIG.apiMode` 改为 `ApiMode.MOCK`。
- **仅验证码切到 Mock**：保持 `apiMode = ApiMode.REMOTE`，将 `APP_CONFIG.verification.requestMode` 改为 `ApiMode.MOCK`。
- **切到开发态本地联调**：将 `DEFAULT_APP_ENVIRONMENT` 改为 `AppEnvironment.DEVELOPMENT`，`apiBaseUrl` 会自动切回 `http://localhost:3001`。
- **关闭万能验证码**：将 `APP_CONFIG.verification.enableUniversalCode` 改为 `false`。
- **禁止远程失败后自动回退 Mock**：将 `APP_CONFIG.verification.allowMockFallback` 改为 `false`。

> 正式环境下，即使误将 `enableUniversalCode` 或 `allowMockFallback` 设为 `true`，万能验证码与 Mock 回退也不会生效，受 `isDevelopmentEnvironment()` 闸门保护。

### 5.5 权限说明

`apps/harmony/entry/src/main/module.json5` 中 `requestPermissions` 声明了两个权限：

| 权限名 | 用途 | 说明 |
| --- | --- | --- |
| `ohos.permission.INTERNET` | 网络访问 | 调用 `https://api.youziyi.com`（正式）或 `http://localhost:3001`（本地联调）所需 |
| `ohos.permission.MICROPHONE` | 麦克风录音 | 用于后续语音接龙等录音场景，`usedScene.abilities = [EntryAbility]`，`usedScene.when = inuse`（仅使用时授权），`reason = $string:microphone_reason` |

首次运行涉及录音功能时，系统会弹出麦克风授权弹窗，用户授权后方可采集音频；网络权限在 HAP 安装时默认授予。

## 6. 常见问题解决

### 6.1 签名配置失败 / 证书缺失

**现象**：构建报 `signing config not found`、`certificate file not exist` 或安装时报 `signature verification failed`。

**排查**：

1. 打开 `build-profile.json5`，确认 `signingConfigs[0].name = default` 与 `products[0].signingConfig = default` 一致。
2. 确认 `material.certpath`、`material.profile`、`material.storeFile` 指向的文件在本机真实存在；若不存在，请通过 DevEco Studio `File > Project Structure > Signing Configs` 勾选 `Automatically generate signature` 重新生成本机调试证书。
3. **禁止**直接复用仓库中他人证书或硬编码密钥口令；调试证书必须本机生成。

### 6.2 SDK 版本不匹配

**现象**：打开工程报 `compatibleSdkVersion 5.0.0(12) not found` 或 `targetSdkVersion 6.0.2(22) not found`。

**排查**：

1. 在 DevEco Studio `File > Settings > SDK Manager` 中确认已下载 API 12（对应 `5.0.0(12)`）与 API 22（对应 `6.0.2(22)`）。
2. 若本机仅能下载到 API 12，请向工程负责人确认是否临时下调 `targetSdkVersion`；禁止在未评估兼容性的情况下私自改为其他版本。
3. 修改 SDK 版本后需执行 `Sync and Refresh Project` 让 hvigor 重新解析。

### 6.3 oh_modules 安装失败 / 同步超时

**现象**：`ohpm install` 报 `request timeout`、`registry unreachable` 或 `@ohos/hypium@1.0.19 not found`。

**排查**：

1. 检查网络是否能访问 ohpm 官方源；公司内网环境下请配置代理或内部镜像。
2. 在 `~/.ohpm/.ohpmrc` 中设置 `registry=https://ohpm.openharmony.cn/ohpm/`（或可用的镜像源）。
3. 删除 `apps/harmony/oh_modules`、`apps/harmony/entry/oh_modules` 与 `oh-package-lock.json5`（如存在）后重新执行 `ohpm install`。
4. 确认 `@ohos/hypium` 版本固定为 `1.0.19`、`@ohos/hamock` 版本固定为 `1.0.0`，不要手动改成不存在的版本号。

### 6.4 网络权限（INTERNET）未生效

**现象**：运行时接口请求报 `Permission denied` 或 `Network request failed`。

**排查**：

1. 确认 `module.json5` 中 `requestPermissions` 包含 `ohos.permission.INTERNET`（本工程已声明）。
2. 检查 `AppConfig.ets` 中 `apiBaseUrl` 是否使用了 `http://` 明文地址；正式环境下默认 `https://api.youziyi.com`，本地联调 `http://localhost:3001` 需在 `development` 环境下使用。
3. 真机需保持网络可达；模拟器需确认宿主机网络正常。

### 6.5 麦克风权限（MICROPHONE）未授权

**现象**：录音相关功能无响应或报 `mic permission denied`。

**排查**：

1. 首次触发录音时系统会弹授权窗，请引导用户选择「允许」。
2. 若用户误选「拒绝」，可在「设置 > 应用管理 > 游子衣 > 权限管理」中重新开启麦克风权限。
3. 确认 `module.json5` 中 `MICROPHONE` 的 `usedScene.abilities` 包含 `EntryAbility`、`when = inuse`，且 `reason = $string:microphone_reason` 已在 `resources/base/element/string.json` 中定义。

### 6.6 远程接口连通失败

**现象**：登录页请求 `GET /api/health` 失败，或业务接口报 `ECONNREFUSED`、`api.youziyi.com 不可达`。

**排查**：

1. **正式环境**：确认 `AppConfig.ets` 中 `DEFAULT_APP_ENVIRONMENT = AppEnvironment.PRODUCTION`，`apiBaseUrl` 应为 `https://api.youziyi.com`；用浏览器或 `curl` 验证 `https://api.youziyi.com/api/health` 是否可达。
2. **本地联调**：确认 `DEFAULT_APP_ENVIRONMENT = AppEnvironment.DEVELOPMENT`，`apiBaseUrl` 应为 `http://localhost:3001`；后端服务需启动并监听 `3001` 端口（与 Web 代理、小程序请求基地址保持一致）。
3. **ECONNREFUSED**：依次检查后端服务是否启动、是否监听 `3001`、环境变量与代理是否一致、是否硬编码了旧端口或旧主机地址；若近期调整过后端端口，必须同步更新 `AppConfig.ets`、Web 代理、小程序请求基地址与 `docs/API接口规范.md`。
4. 若仅鸿蒙端报 `ECONNREFUSED` 而其他端正常，优先排查鸿蒙端独有的 `AppConfig.ets` 配置与本机网络（如模拟器无法访问宿主机 `localhost`，需改用宿主机局域网 IP）。

### 6.7 本地联调端口（3001）

**现象**：本地联调时所有接口 502 / 超时。

**排查**：

1. 确认后端服务已启动且监听 `3001` 端口（与《技术开发规则》一致，禁止私自改端口）。
2. 真机联调时，`http://localhost:3001` 中的 `localhost` 指代真机本身而非开发机；需将 `LOCAL_API_BASE_URL` 临时改为开发机局域网 IP（如 `http://192.168.x.x:3001`），且开发机防火墙需放行 `3001` 端口。
3. 模拟器联调时，部分 HarmonyOS 模拟器可通过 `10.0.2.2` 访问宿主机，请按模拟器文档确认网关映射后再调整 `LOCAL_API_BASE_URL`。
4. 端口调整后必须同步更新 Web 代理、小程序请求基地址、启动脚本与 `docs/API接口规范.md`，禁止仅改鸿蒙端绕过统一约定。

### 6.8 HVIGOR 构建报错

**现象**：`hvigorw assembleHap` 报 `task failed`、`plugin not found` 或 `modelVersion mismatch`。

**排查**：

1. 确认 DevEco Studio 版本与 `hvigor-config.json5` 中 `modelVersion = 5.0.0` 兼容；过旧的 DevEco Studio 可能不支持该 modelVersion。
2. 确认 `hvigorfile.ts` 引入的 `@ohos/hvigor-ohos-plugin` 已随 IDE 正确安装；若 `oh_modules` 缺失，先执行 `ohpm install`。
3. 查看 `hvigor` 详细日志：在命令后追加 `--info` 或 `--debug` 获取完整堆栈。
4. 缓存损坏时，可删除 `apps/harmony/.hvigor` 缓存目录后重新构建。

### 6.9 HAP 安装失败

**现象**：`hdc install` 报 `install failed`、`signature verify failed`、`version mismatch`。

**排查**：

1. **签名不匹配**：HAP 签名证书与设备已安装旧版本不一致；先执行 `hdc uninstall com.youziyi.harmony` 卸载旧版，再重新安装。
2. **版本号回退**：新 HAP 的 `versionCode` 必须大于等于设备已安装版本（当前 `versionCode = 1000000`）；若需降级，先卸载旧版。
3. **架构不匹配**：确认 HAP 构建目标设备类型在 `phone/tablet/2in1` 范围内。
4. **权限原因**：部分设备需在「开发者选项」中打开「禁止权限监控」或「USB 安装应用」开关，否则 HAP 安装会被系统拦截。

### 6.10 模拟器启动失败

**现象**：DevEco Studio Device Manager 中模拟器启动卡住、黑屏或秒退。

**排查**：

1. 确认本机 CPU 支持硬件虚拟化（VT-x / AMD-V），BIOS 中已开启虚拟化。
2. 确认 DevEco Studio 模拟器镜像已完整下载；可在 Device Manager 中删除旧镜像后重新下载。
3. 确认本机内存 ≥ 8GB（推荐 16GB），模拟器需占用较多内存。
4. 若模拟器反复无法启动，可改用真机调试（参考 4.2 节）作为替代方案。
