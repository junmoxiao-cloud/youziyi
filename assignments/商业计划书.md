# 游子衣 - 商业计划书

## 1. 项目概述

“游子衣”是一款面向异地子女与空巢老人家庭的亲情连接与智能关怀服务，核心定位是通过轻量互动、状态同步、风险预警和家庭记忆沉淀，降低异地家庭的情感断联与安全焦虑。

本项目当前以课程答辩和 MVP 验证为目标，不追求一步到位的大规模商业化，而是优先证明产品价值、用户接受度和合规可行性。

## 2. 商业逻辑

### 2.1 要解决的问题

目标用户面临的核心痛点包括：
- 子女长期异地求学或工作，日常联系频率低，容易产生情感愧疚。
- 老人对复杂 App 不友好，常规社交工具难以形成稳定互动习惯。
- 家庭成员担心老人突发状况无法被及时发现。
- 传统通讯方式能“联系”，但难以“沉淀关系”和“建立持续陪伴”。

### 2.2 产品价值

“游子衣”的价值分为三层：
- **情感价值**：让问候从低频、高门槛的电话，变成高频、低门槛的轻量互动。
- **安全价值**：通过打卡、状态同步和超时预警，降低失联风险。
- **记忆价值**：通过语音接龙和故事沉淀，把亲情内容从即时沟通转化为长期保存的家庭数字资产。

### 2.3 商业闭环

产品先通过高频轻互动建立留存，再通过增值服务实现转化：
1. 免费功能吸引家庭用户开始使用。
2. 持续互动形成数据与情感沉淀。
3. 在高价值场景中提供付费服务，如健康报告导出、语音故事长期存储、家庭高级会员。

## 3. 目标用户

### 3.1 核心用户

- **子女端用户**：18-35 岁，异地求学或工作，与父母分居两地，有稳定智能手机使用习惯。
- **老人端用户**：55 岁以上，居家或空巢，愿意接受简单数字工具，但无法适应复杂交互流程。

### 3.2 场景用户

- 社区工作者、邻里联系人，可在高等级预警场景中承担线下介入角色。
- 高校学生团队、社区志愿组织，可作为 MVP 公测和场景验证的首批合作对象。

## 4. MVP 公测策略

### 4.1 MVP 范围

首阶段只验证最核心的三件事：
- 老人是否能完成低门槛打卡。
- 子女是否愿意持续查看和反馈。
- 预警机制和情感化功能是否能形成差异化价值感知。

因此 MVP 优先开放：
- 心情与步数打卡。
- Web 大屏状态查看。
- 基础预警查询。
- 城市天气瓶展示。
- 语音接龙的基础录制与展示能力。

### 4.2 公测路径

大学生团队的现实路径应当“小范围、低风险、可回收”：
1. 先在校园内、亲友家庭、社区联系点中进行小规模试用。
2. 使用测试账号和演示环境验证交互流程，避免一开始就进入开放注册。
3. 优先测试非敏感功能和低风险交互，例如打卡、天气瓶、语音故事。
4. 在形成稳定使用反馈后，再逐步扩展到更多家庭样本。

### 4.3 指标设计

MVP 阶段重点关注：
- 首次打卡完成率。
- 一周内家庭互动次数。
- Web 端查看频率。
- 语音接龙参与率。
- 预警提醒后的响应时间。

这些指标优先服务于“产品是否有用”的判断，而不是立即服务于营收。

## 5. 数据隐私与弱实名策略

### 5.1 弱实名原则

“游子衣”不以强实名为前提。课程项目和 MVP 阶段采用弱实名策略：
- 用户可以使用昵称而非真实姓名。
- 家庭关系通过邀请码、绑定码或约定身份建立，而不是公开搜索。
- 手机号仅在登录、找回和紧急通知场景下使用，不作为公开展示信息。

### 5.2 最小必要采集

只采集维持功能所需的数据：
- 心情枚举值。
- 步数、可选心率等健康状态数据。
- 语音片段及其基础元信息。
- 紧急联系人信息。

默认不采集：
- 通讯录。
- 精确定位。
- 相册。
- 与课程目标无关的敏感身份信息。

### 5.3 数据保护策略

- 生产环境使用 HTTPS 传输。
- 不同家庭间数据隔离，子女仅能访问绑定家庭的数据。
- 语音数据后续迁移至对象存储，并开启访问控制和加密能力。
- 支持账号注销与历史数据清理，遵循最小必要和可撤回原则。

## 6. 盈利模式与发展路径

### 6.1 阶段一：价值验证

当前阶段以答辩和 MVP 公测为主，核心目标是：
- 证明老人端极简交互可行。
- 证明子女端有持续查看和反馈意愿。
- 证明“情感连接 + 安全守护”的组合价值成立。

### 6.2 阶段二：轻量收费

在产品稳定后，可尝试家庭订阅模式：
- 高级会员：更多家庭成员接入、语音存储扩容、健康趋势导出。
- 家庭纪念册：语音故事整理、年度亲情回顾。
- 个性化提醒：针对慢病老人家庭的定制提醒模板。

### 6.3 阶段三：生态合作

未来可探索与以下对象的合作：
- 社区服务中心。
- 养老服务机构。
- 基层医疗或慢病管理服务。
- 智能硬件厂商。

合作方向不是直接替代线下服务，而是成为连接家庭、社区和服务资源的数字入口。

## 7. 长远发展路径

“游子衣”的长期发展可以分为三步：
1. **课程项目 -> 校园 MVP**：完成最小功能闭环和答辩交付。
2. **校园 MVP -> 社区试点**：与真实家庭和社区场景建立小规模试用。
3. **社区试点 -> 服务平台**：逐步从亲情互动工具发展为兼顾情感记录、风险预警和家庭照护协同的平台。

从产品演进看，后续重点包括：
- 接入真实天气 API 和更稳定的消息能力。
- 完整打通预警通知链路。
- 引入对象存储和更规范的数据治理。
- 丰富家庭记忆沉淀能力，如语音时间轴、家庭播客和年度回顾。

## 8. 当前结论

“游子衣”在商业上不是一个追求短期爆发的流量产品，而是一个围绕高频轻互动、长期情感沉淀和风险守护构建的家庭服务型产品。

对于大学生团队而言，最合理的路线不是直接大规模上线，而是先完成：
- 小范围真实场景验证。
- 弱实名与最小采集的数据合规方案。
- 低成本、可复用的技术部署方案。
- 足够支撑答辩和后续试点的文档交付体系。

在这个前提下，项目既能满足课程目标，也具备继续演进为真实产品的可能性。
